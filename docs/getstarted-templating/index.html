üìÑ Templating system

Tideflow template system lets you customise your workflow's tasks data. Instead of hardcoding what a task is expected to do, tasks can take data from previous tasks results.

Let's put a example: you want to create a PDF file, but it needs to contain the name of a user, which is provided by the workflow's endpoint trigger: an endpoint.

Instead of hardcoding the user's name in the PDF template, your template can take the user's name and other details provided by the endpoint's POST body.

Post request example:

curl --location --request POST 'http://localhost:3000/service/endpoint/2cc6ce80-8235-4cbc-b72f-9dbc3cf8364d' \
--header 'Content-Type: application/json' \
--data-raw '{
  "name": "Jose",
  "location": {
      "lat": 42.8867647,
      "lon": -9.2716399
  }
}'

PDF Template

Hello {{trigger.result.data.name}}! 

We found you. You are in:
{{trigger.result.data.location.lat}} & {{trigger.result.data.location.lon}}

Where to use it

Tideflow integrates its templating system in most of the places where it could be used.

Aditionally, you will see, via the workflow editor, every place where it can be used.

How to use it

Using the templating system is as easy as placing square brackets {{}} whenever your want Tideflow to replace it with constom values.

For example, if you want to use a value provided by the workflow's trigger, you can do so as:

{{trigger.result.data.SomeValue}}

This will take the property "SomeValue" from the trigger results.

Apart from the trigger's data, you can use any task output. For example:

{{MyTaskId.result.data.SomeOtherValue}}

This will take the property "SomeOtherValue" from the task which id is "MyTask".

You can customise each task's id via the workflow editor.

How it works

Tideflow's templating system uses Handlebars. This lets you use simple expressions, nested properties, context evaluations, conditionals, etc.

Simple expressions

Replacing your template with the corresponding properties:

{ "name": "Jose", "surname": "Constela" }

<p>{{name}} {{surname}}</p>

Nested objects

Input objects contains other objects or arrays:

{
  person: {
    firstname: "Yehuda",
    lastname: "Katz",
  }
}

{{person.firstname}} {{person.lastname}}

Evaluation context

The built-in block-helpers each and with allow you to change the current evaluation context.

The with-helper dives into an object-property, giving you access to its properties

{{#with person}}
  {{firstname}} {{lastname}}
{{/with}}

The each-helper iterates an array, allowing to you access the properties of each object via simple handlebars expressions.

{
  people: [
    "Yehuda Katz",
    "Alan Johnson",
    "Charles Jolley",
  ],
}

<ul class="people_list">
  {{#each people}}
    <li>{{this}}</li>
  {{/each}}
</ul>

Conditionals

Handlebars' built-in if and unless control structures:

if example:

{{#if isActive}}
  <img src="star.gif" alt="Active">
{{/if}}
